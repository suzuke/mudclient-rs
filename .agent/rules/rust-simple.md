# Rust 開發規範 (Simple & Clean)

這份規則遵循「簡單就是美」的原則，旨在保持 Rust 代碼的純粹與高效。

## 1. 核心準則
- **警告即錯誤**：始終保持代碼通過 `cargo clippy` 檢查。嚴禁帶著警告提交代碼。
- **清晰意圖**：優先編寫容易理解的代碼，而非精巧但晦澀的抽象。如果一段代碼需要超過三次解釋，請重寫它。
- **保持慣用 (Idiomatic)**：遵循 Rust API 指南與 `rustfmt` 的風格建議。

## 2. 實踐建議
- **錯誤處理**：
    - 多數應用程序建議使用 `anyhow` 提供上下文。
    - 關鍵邏輯應使用 `Result` 與 `match` 或 `?` 操作符。
    - 盡量避免 `unwrap()`，若確定不會失敗，請使用 `expect("說明理由")`。
- **註解與文檔**：
    - 所有的 `pub` 函數與結構體必須包含簡單的 `///` 註解（使用繁體中文）。
    - 註解應集中於「為什麼」而非「怎麼做」。

## 3. 工具使用
- 優先使用 `uv` 管理 Python 側依賴（如有混合項目）。
- 始終確保 `cargo test` 在交付前通過。
